### Патч 260221
Разработка API ведётся на python-фреймворке fastapi.
На данный момент в проекте реализована jwt-аутентификация через библиотеку AuthX. Пока что она только выдаёт токены пользователя при вводе заданных значений. Пока база данных не будет достаточно готова, она не будет связана с сервисом аутентификации. Связь с базой данных реализована через orm-фреймворк sqlalchemy. Целевая база данных в данный момент - PostgreSQL.

В этом обновлении было внесено только одно, но крайне важное изменение: полная проработка базы данных. В настоящий момент база данных состоит из следующих таблиц:
| Таблица | Назначение |
| ---     |  ---       |
User | Таблица пользователя: хранит всю информацию об активности пользователя внутри платформы. Имеет тиры. Изначально планировалась система, схожая с правами в системе unix, но за отсутствием необходимости в обильном количестве прав было решено упростить систему до простых числовых обозначений ( -1: Ограниченный пользователь; 0: Пользователь; 1: Модератор; 2: Администратор. Далее в этой таблице они будут обозначаться буквой Т и числом)
Ad  | Таблица объявлений. Хранит в себе всю основную информацию об автомобиле, а так же хранит автора объявления.
Review | Таблица отзывов. Каждый пользователь начиная от Т0 может отправлять отзывы на других пользователей. 
Ticket | Тикеты, которые могут создавать все пользователи включая Т-1 для общения с модерацией и администрацией платформы.
Message | Сообщения в рамках тикета. Отправлять сообщения в тикет может только автор сообщения и любой пользователь с Т1 и выше.
